# Contributing

Contributions are welcome.

1. Choose the branch matching the loader and Minecraft version you are changing.
2. Create a focused branch from that target branch.
3. Keep loader- and version-specific code on its matching branch.
4. Run `./gradlew build` (`gradlew.bat build` on Windows).
5. Open a pull request describing behavior changes, compatibility impact, and validation performed.

Do not commit generated run directories, Gradle caches, IDE metadata, or copied Create binaries.
